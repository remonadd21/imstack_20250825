복습문제 시간

1. git에서 변경된 파일을 스테이지에 올리는 명령어는?
> git add .

2. git에서 커밋 메시지를 작성하여 변경사항을 저장하는 명령어는? 
> git commit -m "메세지"

3. GitHub에 로컬 변경사항을 업로드하는 명령어는?
> git push origin master

4. GitHub에서 최신 변경사항을 로컬로 가져오는 명령어는?
> git pull origin master


5. Git과  GitHup의 관계를 설명하시오
> git은 "로컬"에 저장되고, github는 "클라우드" 에서 협업한다.

6. 클라우드와 내 최신 파일이 다를때 생기는 충돌
> Confict (cntent)  : Merge conflict in 파일명

7. 충돌을 해결한 후 저장하는 방식은?
> git add. -> git commit -m "충돌 해결"


8. 병합이 완료되지 않은 상태에서 'git pull' 을 실행하면 어떤 오류가 발생하나? 
> MERGE_HEAD exists '오류' 

9. .gitignore 파일의 역할은 무엇인가?
> 특정 파일이나 폴더를 버전관리 "git" 에서 제외시킨다.

10. .gitignore에 추가하면 git이 추적하지 않은 파일 확장자 두가지를 작성하시오.
(예: *.log, *.DS_Store)

> *.log, *.DS_Store

11. git에서 로컬 파일을 수정한 수 커밋하는 명령어를 순서대로 작성하시오.
> git add . -> git commit -m "충돌 발생 메세지"

12. github에서 파일을 수정한 후 로컬에서 pull을 하면 발생할 수 있는 문제는?
> 충돌 (conflict)  (힌트, 충돌) 

13. 충돌 해결 후 push까지 완료하는 전체 명령어의 흐름을 작성하시오.
1. git add .
2. git commit -m "충돌 해결 메세지"
3. git pull origin master (최신값 가져와서 비교)
4. git push origin master (변경값 확인후 업로드)
